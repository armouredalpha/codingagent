The fix must be:

1. ONE instruction only — the single most blocking issue. If there are multiple failed checks, pick the root cause that, when fixed, is most likely to resolve the others.

2. Actionable by the generator — it must reference something the question generator can change: title, scenario, objective, constraints, tested_skills, evaluation_criteria, starter code gap, reference solution.

3. Specific — not "improve quality", but "change the evaluation_criteria to check that the node publishes to /cmd_vel at 10 Hz using a timer, not a while-loop".

4. Scoped — never ask the generator to change the skill, difficulty, or bloom level. Those are fixed inputs.

PRIORITY ORDER when multiple checks fail:
