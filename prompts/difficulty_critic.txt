EASY — Robotics Intern level. 5–20 min solve time.
  - Single ROS2 concept (one API call group: publisher OR subscriber OR service)
  - One file, one TODO block, <20 lines to write
  - Bloom: apply — the student copies a known pattern into the gap
  - No integration, no debugging, no design decisions

MEDIUM — Junior Robotics Engineer level. 20–45 min solve time.
  - 2–3 related ROS2 concepts that must work together
  - 1–2 files, 20–60 lines to write
  - Bloom: apply-to-analyze — student must understand how parts interact
  - Some integration (e.g. subscriber feeds a service, timer drives a publisher)
  - Light debugging (fix a broken connection, correct a message type)

HARD — Robotics Engineer level. 45–90 min solve time.
  - Multiple ROS2 concepts with cross-cutting concerns (TF + launch + params, or multi-node workflow)
  - Multiple files, >60 lines to write
  - Bloom: analyze-to-create — student must make architecture or design decisions
  - Non-trivial integration; system-level behaviour must be reasoned about

Score each question 0–100 internally across these dimensions:

  bloom_complexity      (0–35)  — Bloom level weight; Evaluate/Create cannot be easy
  concept_complexity    (0–25)  — Number of distinct ROS2 skill areas required
  implementation_effort (0–20)  — LOC + files modified + integration steps
  debugging_complexity  (0–10)  — Is diagnosing broken behaviour required?
  engineering_reasoning (0–10)  — Does the student make design choices?

Mapping: 0–35 → easy  |  36–70 → medium  |  71–100 → hard

OVERRIDE RULES (these beat the score):
  - Bloom = Evaluate or Create  → cannot be easy
  - Requires architecture decision  → must be hard
  - Single TODO block in one file  → cannot be hard
  - Bloom weight always outranks LOC count alone
